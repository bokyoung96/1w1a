## Problem Statement
The current `crypto/` lane has the foundations for domain models, exchange adapters, paper sessions, reporting, promotion, and validation, but it does not yet behave like a true strategy factory. The next system must generate many strategies, evaluate them for robustness and orthogonality, keep the active basket concentrated but diversified, and collapse their signals into one all-in-one portfolio position.

## What Makes This Cool
The factory is no longer just a set of hand-written strategies. It becomes a machine that:
- generates a candidate pool,
- scores candidates on post-cost quality and robustness,
- rejects correlated lookalikes,
- documents each strategy plainly,
- selects a diversified active basket,
- and compacts all of that into one portfolio-level position.

## Locked design choices
### Factory order
1. Parametric factory first
2. Compositional factory second
3. Open-ended invention later

### Candidate generation
- Parametric generation uses a hybrid approach:
  - small fixed grids for coverage
  - adaptive/random expansion around promising regions
- Candidate pool size target: 30-50

### Scoring / robustness / costs
- Primary objective: risk-adjusted return
- Strategy score = performance + robustness
- Robustness = OOS stability + parameter sensitivity + regime stability
- Transaction cost treatment = post-cost score plus explicit turnover/cost penalty
- Multi-frequency signals are allowed and expected

### Orthogonality and basket construction
- Orthogonality is required for stable performance
- V1 orthogonality measure = return correlation + strategy family diversity
- Active basket size = 10 strategies
- Max concentration = 3 strategies per family
- Allocator weighting = hierarchical
- Top-level allocator buckets = strategy family

### Time / execution behavior
- Strategies may operate on mixed internal cadences
- Portfolio allocator master cadence = event-driven
- Event trigger rule = hybrid:
  - meaningful score change
  - regime change
  - orthogonality constraint breach
- Staged / split execution is required for certain events or price conditions

### Documentation requirement
Every strategy gets its own document under:
- `crypto/strategies/docs/<strategy-name>.md`

Each doc must explain:
- what the strategy is
- how it works
- the economic rationale

### Strategy family library target
Keep the existing 5 families and expand to 8-10 in v1.

Current 5:
1. trend-following breakout
2. mean reversion
3. perp momentum / relative-strength rotation
4. funding-rate carry / funding-aware filter
5. volatility regime / breakout confirmation

Approved added families:
6. trend pullback continuation
7. short-term reversal / exhaustion fade
8. volume / participation imbalance
9. basis / spread dislocation proxy
10. market structure / support-resistance reaction

## Architecture
The strategy factory should extend the existing `crypto/` package instead of creating a second factory stack elsewhere.

Recommended package directions:
- `crypto/strategies/` remains the home for family definitions and registration
- add a candidate-generation layer for parametric factories
- add a strategy-scoring / ranking layer for post-cost + robustness scoring
- add an orthogonality / selection layer for basket formation
- add an allocator layer for 10-strategy all-in-one position logic
- keep paper and reporting consumers downstream of the selected basket

## Constraints
- No real-money execution
- Keep the `crypto/` boundary separate from stock backtesting
- Do not let one family dominate the basket
- Avoid overfitting theater; robustness and orthogonality are hard requirements
- Graphs should be shown from the reporting layer once the allocator has outputs worth showing

## Success Criteria
A first version succeeds if it can:
- define 8-10 families,
- generate 30-50 candidate strategies,
- score them with post-cost + robustness logic,
- filter them with orthogonality constraints,
- choose 10 active strategies,
- weight them hierarchically by family,
- and output one all-in-one portfolio position plus graph-ready results.
